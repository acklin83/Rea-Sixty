# Building the user manual PDF

`docs/user-manual.md` is markdown with pandoc front-matter (TOC, geometry, fonts) ready for `pandoc` → PDF.

## Quick path

```bash
./docs/build-manual.sh
```

Outputs `docs/user-manual.pdf`.

## Pre-requisites

- **pandoc** — `brew install pandoc` (macOS) / `sudo apt install pandoc` (Linux).
- **A LaTeX engine.** The script prefers `tectonic` (self-contained, no system LaTeX install required); falls back to `xelatex` from a TeX distribution.
  - macOS, lightest: `brew install --cask basictex` then re-source PATH (`eval "$(/usr/libexec/path_helper)"`).
  - Cross-platform alternative: `cargo install tectonic`.
  - Linux: `sudo apt install texlive-xetex texlive-fonts-recommended`.
- **Lato font** for the PDF body (full Unicode coverage including the `→` arrow). On macOS Lato is usually already installed; if not: download from <https://fonts.google.com/specimen/Lato>.

## What the script does

`pandoc docs/user-manual.md -o docs/user-manual.pdf` with:

- `--toc --toc-depth=3` — autogenerated table of contents
- `--pdf-engine=tectonic` (or xelatex fallback)
- `-V mainfont="Lato" -V sansfont="Lato" -V monofont="Menlo"`
- Coloured links + black TOC entries

The `\newpage` directives in `user-manual.md` force section breaks. Page geometry, font size, and line spacing live in the YAML front-matter at the top of the markdown.

## Why Lato

Helvetica on macOS lacks U+2192 (`→`) — pandoc + xelatex emits ~80 "missing character" warnings and the rendered PDF has blanks where arrows should appear. Lato is a free Google font with full coverage that we ship the configuration for.
